# План: автоматизированный флоу публикаций в Instagram (Sepia Software)

> Рабочий документ. Цель — от пустого профиля до конвейера «ИИ генерирует пост → ты
> одобряешь в Telegram → система публикует по расписанию». Стек генерации:
> **Anthropic** (тексты) + **Google Gemini/Veo** (картинки/видео). Видео-ядро — твои
> скринкасты; Veo — для faceless/охватных роликов и обложек.

## 0. Зафиксированные решения

- **Провайдеры:** Claude (тексты постов, подписи, сценарии, классификация) — уже подключён;
  Google **Gemini** (Nano Banana / Imagen) — картинки; Google **Veo** — видео. Ключ Google уже в
  `.env.local` как **`NANO_BANANA_KEY`** (один ключ Gemini API даёт и картинки, и Veo). Если Veo по
  качеству не вытянет, добавим Replicate/fal (Kling/Runway/Luma) — архитектура провайдер-агностик.
- **Видео:** основной формат — твои скринкасты Cursor (их API не генерит). Автоматизация для
  них = публикация + авто-обложка + подпись. Veo — опционально для охватных/развлекательных
  рилсов (пост №10 «ожидание vs реальность») и анимированных фонов.
- **Автономность:** **генерация → твоё одобрение → публикация.** Нужен аппрув-шлюз. Используем
  уже подключённый Telegram-бот: расширяем с односторонних алертов до inline-кнопок ✅/✏️/❌.
- **Языки:** старт — **русский** (ru.sepia.software). Дальше — **английский** и **испанский**,
  каждый со своим (адаптированным, не дословно переведённым) контентом. См. §3.5 — инфраструктуру
  делаем язык-агностичной с первого дня, чтобы EN/ES подключались без переписывания.

## 2. Этап 0 — Оформление профиля (старт, делаем первым)

> ⚠️ Поля профиля (имя, bio, аватар, категория, кнопка) **через Graph API не задаются** —
> это только вручную в приложении Instagram. Моя часть — подготовить точные тексты и ассеты,
> твоя — вставить их в приложении. Что можно через API — публикация постов и Stories.

**2.1. Текст профиля (свести расхождения брифа и instagram-profile.md — см. §6):**
- Username: финализировать `@sepia.software` (или свободный близкий).
- Имя (поле Name, индексируется): `ИИ-разработка | Менторство`.
- Bio (≤150 симв.) — взять вариант из брифа (актуальная формулировка «2–3 недели», без слова
  «продакшн»). Я подготовлю финальный вариант после твоего выбора username.
- Категория: «Программист» / «Образование».
- Ссылка в шапке: решить — страница `/links` на ru.sepia.software или Linktree-подобный агрегатор
  (сайт + cal.com/sepia/intro + Telegram-бот).
- Перевести в Business/Creator аккаунт (нужно для API публикации — вероятно уже сделано, проверить)
  → добавить кнопку «Забронировать» → cal.com.

**2.2. Ассеты (генерю по бренд-гайду §3 брифа: тёмный #0a0a0f, неон синий→фиолетовый→зелёный, Inter):**
- Аватар: логотип Sepia на тёмном фоне в градиентной рамке (из `sepia-blue.png`) — рендер по шаблону.
- 8 обложек Highlights в едином неоновом стиле: Менторство · Доводка/аудит · Разработка ·
  Автоматизация · Кейсы · Отзывы · О студии · FAQ.

**2.3. Highlights через API — ограничение:** API умеет **публиковать Stories** (`media_type=STORIES`),
но **собрать их в «Актуальное» можно только вручную** в приложении. План: сценарии сторис уже
написаны ([instagram-scripts.md](instagram-scripts.md), [instagram-profile.md](instagram-profile.md)) —
система отрендерит карточки сторис, ты опубликуешь и добавишь в Highlights руками (разово).

**Definition of done этапа 0:** профиль заполнен, аватар стоит, ссылка в шапке работает,
3 хайлайта (Менторство/Доводка/FAQ) собраны, кнопка созвона активна.

## 3. Целевая архитектура конвейера

```
                 ┌──────────────────────────────────────────────┐
   Контент-план  │  1. ГЕНЕРАЦИЯ (локально, npm run draft)        │
   (12 постов,   │  • Claude → тексты слайдов / подпись / #теги   │
    рубрики)  ─► │  • Шаблон HTML→PNG → слайды карусели/обложки   │
                 │  • Gemini → фоны/иллюстрации (где нужно)        │
                 │  • Veo → faceless-видео (опц.) | твой скринкаст │
                 │  • ассеты → Sanity CDN, черновик → очередь      │
                 └───────────────────────┬──────────────────────┘
                                         ▼
                 ┌──────────────────────────────────────────────┐
   Telegram-бот  │  2. АППРУВ                                     │
   (inline       │  бот шлёт превью (альбом + подпись) + кнопки   │
    кнопки)   ◄──┤  ✅ Опубликовать  ✏️ Правка  ❌ Отклонить      │
                 │  /api/telegram ловит callback → метит черновик │
                 └───────────────────────┬──────────────────────┘
                                         ▼ approved
                 ┌──────────────────────────────────────────────┐
   Vercel Cron   │  3. ПУБЛИКАЦИЯ (по слотам расписания)          │
   (always-on)   │  Graph API: фото / карусель / Reel / Story     │
                 │  лимит Meta: ≤25 постов / 24ч                  │
                 └──────────────────────────────────────────────┘
   (Параллельно, уже работает: DM + комментарии 24/7 на Vercel)
```

**Решение по «где публиковать»:** сейчас память проекта фиксирует «локалка публикует». Для
*расписания + автономности* публикацию логично перенести на **Vercel Cron** (always-on, читает
одобренную очередь из Upstash). Аппрув уже защищает от ошибочных постов. Это сознательное изменение
архитектуры — обновлю память проекта, когда согласуем. Локальный `publish.ts` остаётся как ручной
запасной путь.

## 3.5. Мультиязычность (RU → EN → ES)

Старт — русский, но EN и ES запланированы со **своим контентом**, а не машинным переводом. Чтобы
не переписывать конвейер под каждый язык, закладываем `lang` как измерение **с первого дня**, хотя
живёт пока только RU.

**Ключевое решение — отдельный аккаунт на язык.** docs прямо требуют «EN — отдельный профиль, не
смешивать» ([instagram-profile.md](instagram-profile.md)). Та же логика для ES. Итог: 3 IG-аккаунта
(`@sepia.software` RU, EN, ES), общая кодовая база, конфиг с ключом по языку.

**Что это меняет в архитектуре (закладываем сразу, реализуем по мере запуска языка):**
- **Токены и аккаунты:** хранилище токена сейчас одно-аккаунтное (`ig:token`,
  [lib/store.ts](../lib/store.ts)). Делаем ключи по аккаунту: `ig:token:{lang}`, `IG_USER_ID` →
  per-account. RU работает как раньше; EN/ES добавляются конфигом, а не переписыванием.
- **Бренд-голос и факты:** [lib/brand.ts](../lib/brand.ts) → локали. EN опирается на sepia.software
  (англ. позиционирование уже есть), ES пишем заново в тоне бренда. Claude **генерит нативно** на
  целевом языке (идиомы, а не перевод), хэштеги — свои под локаль (#vibecoding/#nocode для EN,
  #programación/#inteligenciaartificial для ES).
- **Шаблоны визуалов** (M1) — язык-агностичны: текст инжектится, картинка одна и та же. Это главный
  выигрыш — рендер-движок и Gemini-фоны переиспользуются на всех языках без изменений.
- **Контент-план** (M2): `content/plan.{lang}.json`. Темы общие, тексты/примеры/CTA — под рынок.
- **Аппрув и публикация** (M4–M5): черновик несёт `lang`; Telegram-превью помечает язык; cron
  публикует в нужный аккаунт по `lang`. Один Telegram-бот обслуживает все три аккаунта.

**Практический порядок:** RU доводим до рабочего конвейера end-to-end. EN/ES — это «дублирование
конфига + новые тексты», а не новая стройка. Профиль/хайлайты/стартовая сетка повторяются для EN и
ES отдельными прогонами Этапа 0.

## 4. Поэтапный план реализации

### M1 — Бренд-движок визуалов (фундамент для всего)
- `lib/brand-tokens.ts`: цвета, шрифт Inter, градиенты в одном месте.
- `lib/render.ts`: рендер PNG из HTML/CSS-шаблона (satori/@vercel/og или Playwright — Playwright
  уже доступен в окружении). Шаблоны: слайд карусели, обложка рилса, иконка-обложка Highlight,
  карточка-цитата/стат.
- Выход: `npm run render -- <template> <data.json>` → PNG 1080×1350 (карусель) / 1080×1920 (рилс/стори).

### M2 — Генерация текстов (Claude)
- `content/plan.json`: 12 стартовых постов из [instagram-content-plan.md](instagram-content-plan.md)
  как структура (id, формат, рубрика, тема, статус, слот). Сценарии-рыба из
  [instagram-scripts.md](instagram-scripts.md) — как seed.
- `lib/content.ts`: `generatePost(topic, rubric, format)` → структурный вывод (forced tool, как уже
  сделано в [lib/claude.ts](../lib/claude.ts)): `{ slides[], caption, hashtags[], cover_text }`.
  Опирается на [lib/brand.ts](../lib/brand.ts) — никаких выдуманных цен/кейсов.

### M3 — Картинки и видео через API
- `lib/gemini.ts`: генерация картинки (фон/иллюстрация) и видео (Veo). Картинку с **текстом** не
  доверяем модели (нестабильно) — текст всегда поверх шаблоном из M1. Gemini даёт атмосферный фон,
  шаблон кладёт неоновый текст.
- Veo: короткие faceless/охватные ролики (мем «ожидание vs реальность», абстрактная заставка).
  Скринкасты — твой файл на вход, система генерит обложку (кадр + текст) и подпись.

### M4 — Очередь черновиков + аппрув в Telegram
- Очередь в Upstash (расширить [lib/store.ts](../lib/store.ts)): `draft:{id}` со статусом
  `drafted|approved|rejected|published` и слотом.
- `app/api/telegram/route.ts`: приём `callback_query` от inline-кнопок (выставить Telegram webhook).
- Расширить [lib/alert.ts](../lib/alert.ts) → `sendDraftForApproval(draft)`: альбом превью +
  кнопки ✅/✏️/❌. ✏️ → правка подписи ответным сообщением.
- Защита: проверка `chat_id` == владелец, секрет в URL вебхука.

### M5 — Публикация + расписание
- Расширить [lib/ig.ts](../lib/ig.ts):
  - **Карусель:** дочерние контейнеры (`is_carousel_item=true`) → контейнер `media_type=CAROUSEL`
    с `children` → publish. Это закрывает 80% контент-плана.
  - **Stories:** `media_type=STORIES` (для авто-наполнения хайлайтов).
- `app/api/cron/publish/route.ts` (Vercel Cron): берёт `approved` посты, чей слот настал →
  заливает ассеты в Sanity CDN → публикует. Лимит ≤25/24ч.
- Расписание слотов: 2 рилса + 1 карусель в неделю (ритм из контент-плана).

### M6 — Эксплуатация
- `npm run draft:batch` — пакетная генерация недельной пачки (батчинг из контент-плана).
- Позже: сбор метрик (Insights API), авто-предложение тем по тому, что заходит.

## 7. Ограничения и риски (важно понимать заранее)

- **Профиль не настраивается через API** — имя/bio/аватар/категория/кнопка только вручную. Highlights
  тоже собираются руками (API лишь публикует Stories).
- **Лимит Meta: 25 публикаций / 24 часа** (rolling). Нашему ритму (3/нед) с огромным запасом хватает.
- **Текст на AI-картинках ненадёжен** — поэтому весь текст рендерим шаблоном, ИИ-картинка только фон.
- **Скринкасты API не генерит** — это твоя реальная работа в Cursor. Veo закрывает только
  faceless/атмосферу; качество Veo проверим на 2–3 пробных роликах, при провале — Replicate/fal.
- **Веб-аккаунт Telegram-бота** для аппрува: callback-вебхук должен быть публичным (Vercel) и
  защищён секретом, иначе кто угодно сможет жать «опубликовать».
- **Медиа должно быть по публичному URL** для Graph API — уже решено через Sanity CDN.
